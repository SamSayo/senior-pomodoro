### 1. Введение

#### 1.1 Цель
Цель разработки ПО — создание веб-приложения в формате Progressive Web App (PWA) для управления временем по технике Pomodoro. Приложение предлагает три адаптированных режима работы для пользователей с разным уровнем опыта, а также систему достижений для мотивации.

ПО решает следующие задачи:
- повышение личной продуктивности пользователей;
- помощь в поддержании концентрации во время работы;
- формирование привычки регулярных перерывов;
- мотивация через геймификацию (достижения).

#### 1.2 Область применения
Приложение предназначено для использования на любых устройствах с современным веб-браузером (десктопные компьютеры, ноутбуки, планшеты, смартфоны). Благодаря реализации как PWA оно может быть установлено на домашний экран устройства и работать в оффлайн-режиме.

Приложение применяется в повседневной жизни для планирования рабочих/учебных сессий. Основные пользователи — студенты, разработчики, фрилансеры, офисные работники и любые специалисты, желающие улучшить управление временем и повысить продуктивность.

#### 1.3 Объём проекта
В проект входит:
- реализация таймера Pomodoro с тремя предустановленными режимами (junior, middle, senior);
- система достижений и мотивации;
- обеспечение функциональности Progressive Web App (web manifest, service worker, оффлайн-доступ);
- пользовательский интерфейс на базе Next.js и Tailwind CSS.

Не входит в проект:
- пользовательская аутентификация и регистрация;
- серверное хранение данных (синхронизация между устройствами);
- интеграция с внешними сервисами (календари, трекеры задач);
- нативные мобильные приложения (iOS/Android).

#### 1.4 Определения, аббревиатуры и сокращения
- **Pomodoro** — техника управления временем, разработанная Франческо Чирилло, основанная на чередовании интервалов сосредоточенной работы и коротких перерывов.
- **PWA (Progressive Web App)** — прогрессивное веб-приложение, которое сочетает возможности веб-сайта и нативного мобильного приложения (установка, оффлайн-доступ, push-уведомления).
- **Next.js** — фреймворк на базе React для серверного рендеринга и статической генерации веб-приложений.
- **Tailwind CSS** — утилитарный CSS-фреймворк для быстрой разработки интерфейсов.
- **Service Worker** — скрипт, работающий в фоновом режиме браузера, обеспечивающий оффлайн-функциональность и кэширование в PWA.
- **Web Manifest** — JSON-файл, описывающий метаданные PWA (название, иконки, тема).

### 2. Общее описание

#### 2.1 Перспективы продукта
Приложение вписывается в экосистему инструментов повышения продуктивности (Todoist, Forest, Focus Booster и др.), но предлагает упрощённый подход без необходимости регистрации и с адаптированными режимами для разных уровней опыта пользователей.

ПО решает проблемы потери концентрации, прокрастинации и переутомления, предоставляя готовые пресеты длительности сессий и геймифицированную мотивацию через достижения. Как PWA оно обеспечивает кросс-платформенность и доступность без установки из магазинов приложений.

#### 2.2 Функции продукта
Основные функции:
- выбор одного из трёх режимов работы:
– Junior: 15 минут работы / 5 минут перерыва;
– Middle: 25 минут работы / 5 минут перерыва;
– Senior: 45 минут работы / 10 минут перерыва;
- запуск, пауза, остановка и сброс таймера;
- визуальное и звуковое оповещение о завершении сессии работы или перерыва;
- отслеживание завершённых сессий и начисление достижений (например, за количество завершённых помидоров, ежедневные streaks и т.д.);
- отображение статистики и списка разблокированных достижений;
- работа в оффлайн-режиме (благодаря service worker);
- возможность установки как приложения на устройство.

#### 2.3 Характеристики пользователей
Целевая аудитория — пользователи с базовыми навыками работы с веб-интерфейсами (умение открывать сайт в браузере, нажимать кнопки). Специальная подготовка не требуется.

Пользователи обычно интересуются саморазвитием, техниками продуктивности и имеют опыт работы за компьютером/смартфоном. Уровень технических знаний может быть от начинающего до продвинутого; направление — любое (IT, обучение, офисная работа, фриланс).

#### 2.4 Ограничения
- платформа: только веб-браузеры с поддержкой PWA (современные версии Chrome, Edge, Firefox, Safari);
- отсутствие серверной части — все данные хранятся локально (localStorage или IndexedDB);
- ограниченная функциональность уведомлений (зависит от разрешений браузера);
- проект выполнен в рамках курсовой работы, поэтому ограничен по срокам разработки и объёму функциональности;
- нет поддержки старых браузеров (IE и т.п.).

#### 2.5 Предположения и зависимости
Предположения:
- пользователи используют современные браузеры с поддержкой Service Worker и Web Manifest;
- устройство имеет доступ к интернету хотя бы один раз для первоначальной загрузки и кэширования ресурсов.

Зависимости:
- фреймворк Next.js (для структуры приложения и SSR);
- Node.js (для разработки и сборки);
- Tailwind CSS (для стилизации интерфейса);
- браузерные API: Service Worker, Web Manifest, Notifications API, Audio API;
- локальное хранилище браузера для сохранения прогресса и достижений.
